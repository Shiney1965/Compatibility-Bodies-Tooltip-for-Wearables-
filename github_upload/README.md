# Compatible Bodies Tooltip — Source Repository

A mod for Baldur's Gate 3 that adds a "Compatible Bodies" line to the tooltip of each clothing or armor item, showing which body types (BT1 / BT2 / BT3 / BT4) the item is wearable on.

**Published mod (end users):** [Compatible Bodies Tooltip on NexusMods](https://www.nexusmods.com/baldursgate3) (search "Compatible Bodies Tooltip" by SerpentineShel)

This repository exists so that anyone — including antivirus reviewers, NexusMods staff, and curious modders — can read every line of code that goes into the published `.pak`.

---

## Notice about antivirus false positives

Some antivirus products (notably MaxSecure's heuristic engine, label `Trojan.Malware.300983.susgen`) have flagged the published distribution bundle. **`susgen` is MaxSecure's generic "suspicious behavior" detection — it is not a signature match against a known malware family.** Multiple third-party security write-ups describe this specific detection ID as commonly producing false positives on legitimate installer-style files. See `SECURITY.md` for details.

Heuristic false positives on BG3 mod tooling are caused by two things, neither of them malicious:

1. **`divine.exe` from LSLib** — an unsigned third-party .NET binary used to pack and unpack `.pak` archives. **This repository does NOT bundle `divine.exe`.** Built artifacts use Norbyte's LSLib `divine.exe` — source at https://github.com/Norbyte/lslib. Please refer to the upstream project for any binary audit.
2. **`.bat` orchestrators that invoke `.exe` files and copy data** — exactly the behavior pattern heuristic AV engines flag, regardless of intent.

The full source for every Lua, Python, batch, and PowerShell file shipped with the mod is in this repository. None of it does anything malicious. A reviewer can confirm that by reading the code.

## Repository layout

- **`mod/`** — the actual BG3 mod source, as it appears unpacked from `CompatibleBodiesTooltip.pak`.
  - `mod/Mods/AlanTooltipCompat_a7c0ffee-7501-4f00-a17a-c00b08ec4377/ScriptExtender/Lua/BootstrapClient.lua` — main runtime logic. This is the BG3 Script Extender Lua that runs when the player loads the game. It reads from the cache files and updates item description translated-strings to append the compatibility line.
  - `mod/Mods/.../ScriptExtender/Lua/vanilla_cache.lua` — pre-computed compatibility data for vanilla game items, generated by the sidecar pipeline. Pure Lua data, no executable code.
  - `mod/Mods/.../ScriptExtender/Config.json` — Script Extender configuration.
  - `mod/Mods/.../meta.lsx` — Larian mod metadata (UUID, version, dependencies).
  - `mod/Localization/English/AlanTooltipCompat.loca.xml` — localization strings.

- **`sidecar/`** — the build pipeline source. This runs on the modder's machine to generate `vanilla_cache.lua` and any user-cache. It does NOT run inside BG3.
  - `sidecar.py` — main pipeline that walks unpacked BG3 `.pak` data and per-mod RootTemplates to compute body-type compatibility for each item.
  - `run_all.py` / `run_all.bat` — orchestrator that calls `unpack_paks.py` then `sidecar.py`.
  - `unpack_paks.py` — invokes upstream `divine.exe` (from LSLib) to extract `.pak` archives to a working directory.
  - `slim_existing.py` — utility that trims already-extracted `.pak` data to only the files the sidecar needs.
  - `build_pak.bat` — wraps `divine.exe create-package` to pack the mod into a `.pak`.
  - `*.lua` — diagnostic probes (read-only inspectors used during development).
  - `*.bat`, `_pipeline_runner.ps1` — orchestration shell scripts. Plain text, fully readable.

- **`distribution/`** — end-user documentation from the published bundle (README, INSTALL).

- **`LICENSE`** — MIT.
- **`LICENSE-THIRD-PARTY.md`** — attributions for upstream dependencies (LSLib, BG3 Script Extender).
- **`SECURITY.md`** — context on the antivirus false-positive flag.

## How to audit this code

1. Start with `mod/Mods/AlanTooltipCompat_a7c0ffee-7501-4f00-a17a-c00b08ec4377/ScriptExtender/Lua/BootstrapClient.lua` — this is what actually runs inside BG3.
2. The runtime touches only BG3 Script Extender APIs (`Ext.Stats.*`, `Ext.Loca.*`, `Ext.Events.*`). No file I/O outside the mod's own folder, no network, no shelling out, no process spawning.
3. The sidecar (`sidecar/`) is the build-time pipeline. It is invoked by the modder, not by end users. Its only file I/O is reading unpacked BG3 game data and writing two output files: a `compat_cache.json` and a `vanilla_cache.lua`.

## Building from source

Build-time dependencies (not in this repo):
- **Python 3.10+**
- **Norbyte LSLib** — download from https://github.com/Norbyte/lslib/releases. Place `divine.exe` and its DLLs in `sidecar/Tools/`.
- **A Steam-installed copy of Baldur's Gate 3** — the sidecar reads from the game's `Data/` folder.

Then from `sidecar/`:

```bat
run_all.bat
```

This unpacks the base-game `.pak` files, computes compatibility, generates `vanilla_cache.lua`, and packs the mod into `CompatibleBodiesTooltip.pak`.
